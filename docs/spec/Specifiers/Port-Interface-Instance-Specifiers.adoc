=== Port Interface Instance Specifiers

A *port interface instance specifier*
specifies that a
<<Ports_Port-Interface-Instances,port interface instance>> _I_
is part of a topology _T_.
_I_ provides a <<Ports_Port-Interfaces,port interface>>
that can be used in connections in _T_.

==== Syntax

_instance-or-import_
<<Scoping-of-Names_Qualified-Identifiers,_qual-ident_>>

_instance-or-import_ is one of `instance` or `import`.
The syntax `instance` is standard.
The syntax `import` exists for backwards compatibility with previous versions
of FPP.

==== Semantics

The qualified identifier must
<<Scoping-of-Names_Resolution-of-Qualified-Identifiers,refer to>>
a
<<Ports_Port-Interface-Instances,port interface instance>>.

Every port interface instance specifier *represents* a component instance 
definition or topology definition _D_.
A port interface instance specifier _S_ represents _D_
if the port interface instance _I_ to which _S_ refers
<<Ports_Port-Interface-Instances,represents _D_>>.

==== Example

[source,fpp]
----
port P

passive component A { output port p: P }
passive component B { sync input port p: P }

instance a: A base id 0x100
instance b: B base id 0x200

topology T1 {
  instance a
  port a = a.p
}

topology T2 {

  instance b
  instance T1

  connections C {
    T.p -> b.p
  }

}
----
